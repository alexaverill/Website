Accessing Components Children.

While Unit Tests can be extremely helpful, In terms of the QML testing I have done testing individual QML components is pretty much a wasted effort. I have yet to run into a bug in the built in QML components (I may eventually, and will update this page when it happens). Most of the testing I tend towards is combinations of multiple built in QML components to create a more complex custom component. Once a component is created that has a few sub components it can be helpful to use findChild() to access children elements from your base component. findChild() does exactly what you expect and will search a component for a child based on an ObjectName that the programmer defines. ObjectNames can be assigned to any QML component in a similar way to any other properties, and are always a string value.

Example:

Component Code:

import QtQuick 2.0
Rectangle{
    id:background
    objectName:"background"
    color:"black"
    MouseArea{
        id:clickable
        objectName:"clickable"
        anchors.left:parent.left
        anchors.top:parent.top
        width:500
        height:500
        Rectangle{
            id:clickRepresentation
            objectName:"clickRep"
            anchors.fill:parent
            color:clickable.pressed?"green":"yellow"
        }
    }
}

Test Code

import QtQuick 2.0
    import QtTest 1.2
    TestCase{
        id:backgroundTests
        name:"TestBackground"
        when:windowShown
        Background{
            id:bg
        }
        property color expectedBG:"yellow"
        function test__non_clicked_color(){
            let clickRep = findChild(bg,"clickRep");

            compare(clickRep.color,expectedBG);
        }
    }
